1.3 LEAKAGE FINDINGS

[A] CRITICAL: add_swing_detection — center=True look-ahead
    File:      src/features/indicators.py:323-324
    Code:
      df["swing_high"] = high == high.rolling(window * 2 + 1, center=True).max()
      df["swing_low"] = low == low.rolling(window * 2 + 1, center=True).min()
    Problem:  center=True includes FUTURE bars in the rolling window.
              With window=5 (default), looks 5 bars ahead.
    Impact:   27.8% of swing_high and 32.2% of swing_low labels are
              contaminated by future data. These features ARE used in
              model training (in EXTRA_F list, train_v5.py:50).
    Leakage:  100% probability for these 2 features.
    Verdict:  CONFIDENT LEAKAGE DETECTED

[B] MODERATE: BB_pct_b formula mismatch
    File:      src/features/indicators.py:217
    Stored:   Different formula from standard (close-lower)/(upper-lower)
    Impact:   Correlation r≈0.77 vs correct computation (10 symbols).
              Data corruption in stored feature store values.
    Leakage:  0% (not a leakage, but a data quality issue)
    Verdict:  DATA QUALITY ISSUE — stored BB_pct_b is unreliable

[C] MODERATE: Pattern detection uses centered extrema
    File:      src/patterns/chart_patterns.py:48-52
    Code:     argrelextrema with order=window (looks both ways)
    Impact:   Pattern detection (double_top, H&S, etc.) uses future data.
              However, these patterns are stored in pattern_occurrences
              table which is NOT used in model training. Only used for
              backward-looking statistics.
    Leakage:  0% for model training (no effect)
    Verdict:  NOT RELEVANT to prediction pipeline

[D] MODERATE: Regime volatility threshold uses full-history
    File:      src/analysis/regime.py:64
    Code:     vol_threshold = daily["vol_20d"].quantile(0.7)
    Impact:   Full-history 70th percentile determines high_vol labels.
              6% point gap between full-history and expanding window.
    Leakage:  Partial (only volatility_regime column affected)
    Verdict:  CONFIRMED — see Phase 3

[E] LOW: Delivery delta uses same-day delivery_pct
    File:      train_v5.py:107
    Code:     dv['delivery_delta'] = dv['delivery_pct'] - dv['delivery_pct_ma5']
    Impact:   delivery_pct[T] is used as a feature for predicting return(T→T+1).
              Justifiable if prediction runs after market close (delivery data
              released by ~6 PM). Industry standard practice.
    Leakage:  0% if prediction pipeline runs after 6 PM IST
    Verdict:  ACCEPTABLE if timing constraint is met

1.4 Verdict: 2 features out of ~110 have confirmed look-ahead leakage.
   Estimated impact: swing_high/swing_low contribute ~5-15% of model
   predictive power based on their small count in feature set.
   Remediation: Replace center=True with center=False.

  - Compared stored vs correct labels

3.2 Findings

  [A] Volatility regime threshold leakage
      The stored volatility_regime uses full-history quantile(0.7):
        vol_threshold = daily["vol_20d"].quantile(0.7)
      This threshold uses data from ALL dates (including future) to determine
      what "high volatility" means for each date.
      
      Quantification:
        Full-history hv_20 threshold: 17.56
        Stored vs full-history match:  15.23% differ
        Stored vs expanding match:     21.27% differ
        Difference:                     6.04 pp  → FUTURE DATA CONTAMINATION

  [B] Regime label (bull/bear/sideways)
      Uses rolling MA_50 and ret_20d — both computable at time T.
      No look-ahead detected for regime_label and regime_id.
      
  [C] Predictiveness of regime labels (next-day returns):
        bear:     mean=-0.11% std=3.37% n=161
        bull:     mean=+0.11% std=1.50% n=1365
        sideways: mean=+0.07% std=1.58% n=876
        
        high_vol:   mean=+0.14% std=2.23% n=688  (contaminated)
        normal_vol: mean=+0.06% std=1.47% n=1714  (contaminated)

3.3 Impact
  - The regime_id and regime_label (bull/bear/sideways) are clean
  - The volatility_regime is partially contaminated (6% of labels wrong)
  - Regime features are NOT used as model features — they're used for
    post-hoc analysis and regime-conditioned performance reporting.
    Therefore, model predictions are NOT affected.
  - Any strategy that conditions on volatility_regime is invalid.

3.4 Verdict: REGIME LABELS (bull/bear/sideways) CLEAN.
              VOLATILITY REGIME CONTAMINATED (6% look-ahead).
              No impact on core model predictions.

  Method:       Block bootstrap, block=21 trading days, B=5000, M=14 strats
  V_obs:        21.70 (max t-stat, best strategy: t3_net)
  p-value:      0.6060
  Verdict:      FAIL (p>=0.05)

  INTERPRETATION: ALL 14 strategies perform very well (Sharpe 1.4 to 7.5).
  The bootstrap distribution of max t-stats is WIDE because every strategy
  generates high t-stats in each resample. White's RC is testing whether
  the BEST strategy is better than random — but when ALL strategies are
  good, the null distribution is shifted right, making rejection harder.
  This is a KNOWN LIMITATION of White's RC when multiple strong strategies
  exist. The result means: "given that all 14 strategies are strong, we
  cannot say the best one is significantly better than the second-best."
